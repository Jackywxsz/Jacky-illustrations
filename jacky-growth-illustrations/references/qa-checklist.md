# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有 Jacky 小芽哥。
- 身体是白色圆润豆豆/小黑体型。
- 细手细腿清楚。
- 手臂和手清楚；动作图不能缺手。
- 头顶嫩芽清楚可见。
- 有知识工位气质：笔记本、电脑、内容块、浇水壶、路径或行动工具择一即可。
- 如使用墨镜，必须是同款小黑墨镜，小而潇洒，不遮住嫩芽，不显凶。
- 如使用灵感符号，必须是黑色短线式三线，不是彩色圆点三连。
- Jacky 小芽哥承担核心动作，不只是装饰。
- 没有复刻旧小黑案例构图。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 绿色只用于嫩芽或极少量生长点。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- Jacky 小芽哥变成树桩、普通人形男孩、动物、机器人、黑豆、抽象种子或影子怪物。
- 白色豆豆体型不见了。
- 头顶嫩芽缺失、太小或不清楚。
- 墨镜太大、太凶、太油腻、换了款式，或抢走嫩芽识别。
- 灵感符号画成橙/蓝/白彩色圆点三连。
- 手臂或手缺失，身体侧边空白，或手从头顶/嫩芽附近长出。
- 角色像儿童贴纸、表情包或商业 mascot。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。

## 迭代方法

- 太普通：让 Jacky 小芽哥成为动作主体，加入一个奇怪但成立的生长隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太不像潇洒哥：加入小墨镜、松弛站姿、指向动作、跑动动作或单手拿笔记本。
- 墨镜太重：缩小墨镜，保留简单表情，确保嫩芽第一眼可见。
- 墨镜漂移：改回同款小黑墨镜，保留黑色镜片、粗黑框和白色斜高光。
- 灵感符号错误：删除彩色圆点，只保留 3 条黑色短线。
- 缺手：重生成或局部编辑，明确写“两只手臂和两只手可见，除非被道具遮挡但仍露出手势”。
- 太像彩色贴纸：去掉填色和厚描边，只保留白底黑线，嫩芽可少量绿色。
- 太不像参考小芽：强化 white rounded bean/blob body / tiny arms / thin legs / head sprout / knowledge-worker props。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先看到“白色小芽哥 + 头顶嫩芽”，再在 1 秒内看懂结构。

如果第一眼像树桩、普通人、宠物、商业贴纸或凶墨镜角色，就不合格。
