# Jacky 小芽哥无限生长正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是商业插画、PPT 信息图或可爱卡通，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张清爽、怪诞、有创意、可读但不说明书的手绘解释图。

默认视觉 IP 是“Jacky 小芽哥”：一个参考小黑体型改造的白色豆豆角色。核心识别点是圆润白色身体、细手细腿、头顶两片嫩芽、知识工位气质；同款小黑墨镜用来表达“潇洒哥”的从容、自信和行动感，可以正戴，也可以推到头上。Jacky 小芽哥必须参与画面的核心动作，不能只是站在旁边当装饰。

当前固定视觉锚点：

- 白色圆润豆豆身体：头身一体，不要画成树桩、普通男孩或上下分离的人形。
- 头顶绿色双叶嫩芽：这是第一识别点，必须清楚，不能被墨镜或配饰挡住。
- 同款小黑墨镜：黑色镜片、粗黑框、白色斜高光；需要潇洒哥气质时优先使用这一款，不要换成别的眼镜。
- 灵感三线：只画 3 条黑色短线，像突然有想法；禁止画成橙/蓝/白三个彩色圆点。
- 手臂和手：每个动作默认要看得见手臂和手，被道具遮挡时也要露出可识别的手势。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、一鱼多吃、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- Jacky 小芽哥在图里做什么
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景
- 黑色手绘线稿
- 少量红色/橙色/蓝色中文手写批注
- 大量留白
- Jacky 小芽哥作为核心动作主体
- 白色豆豆/小黑体型、头顶嫩芽、细手细腿、知识工位气质
- 同款小黑墨镜可正戴或推到头上，但不能遮住嫩芽，也不能换成别的眼镜
- 动作里的手臂和手必须清楚；不要出现身体侧边空白、手缺失或手从头顶长出来
- 如果出现灵感符号，只能用 3 条黑色短线；禁止橙/蓝/白彩色圆点三连
- 禁止 PPT、商业插画、幼稚可爱、复杂架构、左上角类型标题

不要复刻旧小黑案例。用户参考只提供“白色豆豆体型 + 头顶嫩芽 + 知识工位”的 IP 母体；新图必须是 Jacky 小芽哥的白底黑线版本。每次都要从当前文章重新发明一个奇怪但成立的隐喻。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- Jacky 小芽哥只是装饰
- 头顶嫩芽缺失或不清楚
- 角色变成树桩、普通人形、动物或商业 mascot
- 墨镜太大，抢走嫩芽或表情
- 墨镜款式漂移，不像同一副小黑墨镜
- 灵感符号画成彩色圆点三连
- 动作缺手、缺手臂，或手臂从头顶/嫩芽附近长出
- 画面太满
- 太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“常见坑/流程图/系统架构图”等标题
- 画风太可爱、幼稚、死板
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
